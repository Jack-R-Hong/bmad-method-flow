# Story 25.5: Implement generate-agents-yaml Pack Action

**Epic:** 25 — Agent Mesh Safety Guards
**Story ID:** 25.5
**Status:** done

## Story

As a platform operator,
I want a `generate-agents-yaml` action that produces an `agents.yaml` ACL configuration from the BMAD agent registry,
so that mesh configuration is derived from authoritative definitions and cannot drift.

## Acceptance Criteria

1. **Given** `BmadAgentRegistry` has ACL rules from Story 25.4, **When** the `generate-agents-yaml` action is invoked via `execute_action()` in `src/lib.rs`, **Then** an `agents.yaml` file is written to `config/agents.yaml` (or the path from `agents_yaml_path` config) using `BTreeMap` for deterministic alphabetical key ordering, and the file begins with `# Generated by plugin-coding-pack. Do not edit manually.`.
2. **Given** the generated file is inspected, **When** each agent entry is reviewed, **Then** it contains: `description`, `model` (Claude model string), `max_turns`, `max_budget_usd`, `timeout_secs`, `can_invoke`, `can_respond_to`, `allowed_tools`.
3. **Given** an existing `agents.yaml` already exists, **When** `generate-agents-yaml` is run, **Then** the existing file is overwritten (idempotent generation, no merge).
4. **Given** generation performance is measured, **When** all 9 agents are processed, **Then** the operation completes in <50ms (NFR-AD-7).
5. **Given** the `CodingPackInput` action parser in `src/pack.rs`, **When** `{"action": "generate-agents-yaml"}` is received, **Then** the action is recognized and dispatched to the generate handler, and the result is returned as a success JSON string with the file path and agent count.
6. **Given** unknown or missing workspace config `agents_yaml_path`, **When** `generate-agents-yaml` runs, **Then** it defaults to writing `config/agents.yaml` relative to `config.base_dir`.

### Manifest Path Resolution

The `BmadAgentRegistry::new()` requires a path to the CSV manifest. In `src/lib.rs` (line 55-58), the registry is created with:

```rust
let manifest_path = std::path::PathBuf::from("_bmad/_config/agent-manifest.csv");
let agent_reg = agent_registry::BmadAgentRegistry::new(&manifest_path);
```

In `src/pack.rs`, use the same path pattern. The manifest path is relative to the workspace base_dir, but the registry constructor in `src/lib.rs` uses a hardcoded relative path. Follow the same approach:

```rust
fn generate_agents_yaml(config: &WorkspaceConfig) -> Result<serde_json::Value, WitPluginError> {
    let manifest_path = config.base_dir.join("_bmad/_config/agent-manifest.csv");
    let registry = crate::agent_registry::BmadAgentRegistry::new(&manifest_path);
    // ...
}
```

### WASM Gate

The `agent_registry` module is gated behind `#[cfg(not(target_arch = "wasm32"))]` in `src/lib.rs` (line 2). The `generate-agents-yaml` action uses the agent registry, so it also needs the WASM gate. Two approaches:

1. Gate the `generate-agents-yaml` match arm with `#[cfg(not(target_arch = "wasm32"))]` and return an error for WASM builds.
2. Gate the entire function body.

**Recommendation**: Use approach 1. Add a WASM fallback that returns an error: `"generate-agents-yaml is not available in WASM builds"`.

### BTreeMap for Deterministic Output

Use `BTreeMap` everywhere in the serialized output structure. Never `HashMap`. This ensures alphabetical key ordering in the YAML output:

```rust
use std::collections::BTreeMap;

let mut agents_map: BTreeMap<String, BTreeMap<String, serde_yaml::Value>> = BTreeMap::new();

for agent in registry.list_agents(None) {
    let acl = registry.get_acl(&agent.name);
    let mut entry: BTreeMap<String, serde_yaml::Value> = BTreeMap::new();

    entry.insert("description".to_string(),
        serde_yaml::Value::String(agent.description.unwrap_or_default()));
    entry.insert("model".to_string(),
        serde_yaml::Value::String("claude-sonnet-4-20250514".to_string()));
    entry.insert("max_turns".to_string(),
        serde_yaml::to_value(25).unwrap_or_default());
    entry.insert("max_budget_usd".to_string(),
        serde_yaml::to_value(5.0).unwrap_or_default());
    entry.insert("timeout_secs".to_string(),
        serde_yaml::to_value(300).unwrap_or_default());
    entry.insert("can_invoke".to_string(),
        serde_yaml::to_value(&acl.can_invoke).unwrap_or_default());
    entry.insert("can_respond_to".to_string(),
        serde_yaml::to_value(&acl.can_respond_to).unwrap_or_default());
    entry.insert("allowed_tools".to_string(),
        serde_yaml::to_value(agent.tools.unwrap_or_default()).unwrap_or_default());

    agents_map.insert(agent.name.clone(), entry);
}
```

### Default Agent Field Values

The generated YAML needs concrete values for fields that may not exist on `SdkAgentDefinition`:

| Field | Source | Default |
|-------|--------|---------|
| `description` | `agent.description` | `""` |
| `model` | hardcoded | `"claude-sonnet-4-20250514"` |
| `max_turns` | hardcoded | `25` |
| `max_budget_usd` | hardcoded | `5.0` |
| `timeout_secs` | hardcoded | `300` |
| `can_invoke` | `registry.get_acl()` | from ACL rules |
| `can_respond_to` | `registry.get_acl()` | from ACL rules |
| `allowed_tools` | `agent.tools` | `[]` if None |

Note: The `model` field is hardcoded because `SdkAgentDefinition.model` is always `None` (resolved at runtime). Use a sensible default model string for the YAML file.

### Header Comment

The serde_yaml serializer does not support comments. Prepend the header manually:

```rust
let yaml_body = serde_yaml::to_string(&agents_map)
    .map_err(|e| WitPluginError::internal(format!("YAML serialization error: {e}")))?;
let output = format!("# Generated by plugin-coding-pack. Do not edit manually.\n\n{}", yaml_body);
```

### File Writing

Create parent directories if they don't exist:

```rust
if let Some(parent) = output_path.parent() {
    std::fs::create_dir_all(parent)
        .map_err(|e| WitPluginError::internal(format!("cannot create directory: {e}")))?;
}
std::fs::write(&output_path, &output)
    .map_err(|e| WitPluginError::internal(format!("cannot write agents.yaml: {e}")))?;
```

### Anti-Patterns to Avoid

- Do NOT use `HashMap` anywhere in the serialization chain. Always `BTreeMap`.
- Do NOT merge with existing file content. Always overwrite completely (idempotent).
- Do NOT use `unwrap()` in production code. Map errors to `WitPluginError`.
- Do NOT serialize `AgentMeshSettings` or `AgentAcl` directly. Build the output structure with `BTreeMap` for full control over field naming and ordering.
